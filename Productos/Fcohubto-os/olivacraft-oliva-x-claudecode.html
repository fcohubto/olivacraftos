OLIVA × Claude Code

Guía de ejecución con Claude Code.

Sigue las 8 fases en orden. Cada fase tiene un comando o prompt listo para copiar. El sistema OLIVA opera como copiloto — valida, corrige y mantiene contexto a lo largo de la sesión.

Antes de empezar
Claude Code instalado y autenticado
CLAUDE.md presente en la raíz del proyecto
Proyecto en la carpeta correcta de Olivacraft
Terminal abierta — PowerShell o VSCode
Flujo de trabajo

8 fases. En orden.

No saltes fases. Cada una produce el input que la siguiente necesita.

📂
FASE 01
Abrir workspace

Entra al proyecto desde el terminal. Reemplaza [proyecto] con el nombre de la carpeta.

cd "C:/Users/orell/OneDrive/Olivacraft/Productos/[proyecto]"
Por qué importa.

Claude hereda el contexto desde la carpeta — reglas, estructura y arquitectura del proyecto quedan activos desde el inicio.

Resultado.

Entorno activo con todo el contexto del proyecto disponible.

🧠
FASE 02
Iniciar Claude Code

Levanta Claude dentro del proyecto. El sistema OLIVA se carga automáticamente.

claude
Por qué importa.

Claude lee el CLAUDE.md y absorbe workflows, reglas y filosofía operacional antes de que escribas una sola instrucción.

Resultado.

Claude operando con el contexto completo del sistema OLIVA.

🎯
FASE 03
Definir el objetivo

Escribe qué vas a construir, mejorar o resolver en esta sesión. Una cosa a la vez. El prompt de Fase 04 se actualiza solo.

Tu objetivo para esta sesión
Un objetivo vago produce trabajo vago. Sé específico.
Por qué importa.

Sin un objetivo definido, OLIVA no puede validar ni acotar. Este texto es el input de la Fase 04.

Resultado.

Objetivo claro y verificable antes de ejecutar. Sin ambigüedad.

🔍
FASE 04
Revisión PRE ejecución

Claude valida el objetivo antes de empezar. El prompt de abajo refleja lo que escribiste en Fase 03 — cópialo y envíalo tal cual.

"Voy a trabajar en: [completa la Fase 03 primero]. Revisa si el alcance es claro y acotado. Identifica riesgos o ambigüedades antes de que empiece."
Escribe tu objetivo en Fase 03 para que este prompt se complete solo.
Por qué importa.

OLIVA detecta scope creep y ambigüedad antes de que consuman tiempo. Es más fácil corregir el objetivo que corregir el trabajo.

Resultado.

Objetivo validado. Sabes exactamente qué construir y qué no construir.

⚙️
FASE 05
Construcción guiada

Dale la instrucción de construcción. Claude trabaja bajo las reglas del sistema OLIVA activo.

"Construye [lo que acordamos en Fase 04]. Respeta las reglas del CLAUDE.md activo."
Por qué importa.

El CLAUDE.md garantiza que Claude trabaje con las reglas del proyecto — no inventa patrones propios ni rompe la arquitectura definida.

Resultado.

Implementación consistente con la arquitectura y filosofía del proyecto.

🧪
FASE 06
Auditoría POST ejecución

Pídele a Claude que revise lo que acaba de construir. No saltes este paso.

"Audita lo que acabas de construir. ¿Cumple el objetivo? ¿Introdujo complejidad innecesaria? ¿Qué debe corregirse?"
Por qué importa.

Claude puede detectar errores en su propio trabajo si se lo pides explícitamente. Sin este paso, los problemas llegan directo a deploy.

Resultado.

Trabajo validado antes de publicar. Sin sorpresas en producción.

🚀
FASE 07
Deploy iterativo

Publica los cambios. Usa un mensaje de commit que describa qué se construyó, no cómo.

git add [archivo] && git commit -m "descripción del cambio" && git push origin master
Por qué importa.

Deploy frecuente = feedback real. Los errores se detectan antes y el progreso es visible para todos.

Resultado.

Cambio publicado. GitHub Pages actualiza en 1–2 minutos.

♻️
FASE 08
Continuidad operacional

Antes de cerrar, pídele a Claude que guarde el estado. La próxima sesión arranca desde aquí.

"Resume el estado actual del proyecto y define la siguiente acción recomendada."
Por qué importa.

Sin este paso, cada sesión empieza desde cero. Con él, el proyecto tiene memoria operacional y no se pierde contexto entre sesiones.

Resultado.

La próxima sesión arranca en 30 segundos, no en 15 minutos de re-contexto.

Principio del sistema.

OLIVA no reemplaza tu criterio.

Lo organiza, lo supervisa y lo mantiene consistente mientras Claude acelera la producción. El sistema existe para reducir caos y sostener ejecución real — sesión tras sesión.
